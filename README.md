# Enhancer

Study materials app — no login required for students. Hierarchical structure:
Course/Batch → Subject → Chapter → Lecture.

## Stack
- Flutter (Android + iOS from one codebase)
- Firebase: Firestore (data), Storage (videos/PDFs), Auth (admin only)
- GitHub Actions: builds the release APK automatically on every push to `main`

## Phone-only setup (Termux + Acode + GitHub)

### 1. Create the GitHub repo
1. github.com → New repository → name it `enhancer` → **Public** (needed for free unlimited Actions minutes) → Create.

### 2. Push this project from Termux
```bash
pkg install git -y
git clone https://github.com/<your-username>/enhancer.git
cd enhancer
# copy these project files into this folder using Acode
git add .
git commit -m "Initial Enhancer app scaffold"
git push origin main
```

### 3. Set up Firebase (free)
1. console.firebase.google.com → Add project → name it `enhancer`
2. Enable **Firestore Database** (start in test mode for now)
3. Enable **Storage**
4. Enable **Authentication** → Email/Password (for admin login only)
5. Add an Android app in Firebase project settings, download `google-services.json`, and place it in `android/app/` (once the Flutter android folder exists — see step 4)

### 4. Generate the full Flutter project structure
This repo currently only has `lib/main.dart`, `pubspec.yaml`, and the CI workflow.
Flutter's `android/`, `ios/`, and other platform folders are auto-generated by `flutter create .`
Since you don't have Flutter locally, the easiest phone-only path:
- Use a free cloud IDE like **Gitpod** or **GitHub Codespaces** (opens in browser, works on phone) → run `flutter create .` once → commit the generated folders.

### 5. Every time you push code
GitHub Actions automatically builds the APK. Go to your repo → **Actions** tab → latest run → download the `enhancer-app-release` artifact (a zip containing the APK).

## Data model (Firestore)
```
courses/{courseId}
  name, examType, studentsCount
  subjects/{subjectId}
    name
    chapters/{chapterId}
      name
      lectures/{lectureId}
        title
        youtubeId          # e.g. "dQw4w9WgXcQ" — video stays online-only, no download
        durationSec
        pdfs: [{ name, url, sizeMB, type: notes|dpp|assignment|solution }]  # url = Firebase Storage link, downloadable offline
```

## Decided feature scope
- **Admin panel**: separate web dashboard (its own small project), talks to the same Firebase project. Not built into the mobile app.
- **Videos**: YouTube unlisted links, embedded via `youtube_player_flutter`. Streamed online only — no offline video download.
- **PDFs**: uploaded to Firebase Storage by admin, downloadable offline via Dio + Hive.
- **Search**: course/subject/chapter names + lecture titles.
- **Notifications**: deferred to v2.
- **Monetization**: AdMob banner/interstitial ads.
